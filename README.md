## 📌 About

This project answers specific analytical questions over a dataset, delivering insights in a clean and user-friendly way. The emphasis is on:

- **Object-Oriented Programming (OOP)** - structured, reusable code design
- **Encapsulation** - use of private methods and variables via `_` and `__` conventions
- **Readability** - code that is easy to follow and maintain
- **Performance** - optimized where it matters
- **Concealment** - let users only see results

## 🧠 Key Learnings

- Using `_` prefix to signal **private/internal** methods and variables, making them conventionally protected and easier to refactor in the future
- Overriding **dunder methods** (e.g. `__str__`, `__repr__`) inherited from Python's base `object` class to customize default behavior and improve output readability
